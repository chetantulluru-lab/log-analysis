# 🛡️ Log Analysis (Beginner Cybersecurity Project)

This project demonstrates basic **Linux log analysis** using simple command-line tools.

The goal of this project is to learn how security analysts detect suspicious activity from log files.

---

## 🎯 Objectives

- Understand what logs are
- Identify different types of attacks
- Practice Linux log analysis commands
- Detect suspicious activity from logs

---

## 📂 Project Files

### 📄 log.txt
Sample log file containing:
- Failed login attempts
- Successful logins
- Admin (sudo) activity
- Web attack attempts

---

### 📄 commands.txt
Contains Linux commands used for analysis:

- `grep` → search log entries  
- `sort` → organize results  
- `uniq` → count repeated entries  
- `wc` → count events  

---

### 📄 Analysis_report.txt
Security investigation report including:

- Brute force attack detection
- Suspicious IP identification
- Admin privilege usage
- Web attack attempts

---

## 🔍 Skills Practiced

- Linux terminal basics
- Log investigation
- Attack detection
- Command-line analysis
- Cybersecurity fundamentals

---

## 🧰 Tools Used

- Ubuntu Linux
- Bash Terminal
- grep, sort, uniq, wc

---

## 🚨 Example Detections

- Multiple failed login attempts (Brute Force)
- Successful user login tracking
- Admin command usage monitoring
- Unauthorized admin panel access attempts

---

## 📘 Learning Purpose

This is a **beginner-level cybersecurity practice project** created while learning Linux and Security Operations Center (SOC) basics.
